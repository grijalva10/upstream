# Upstream V2 - Full System Spec

## Overview

Upstream is an autonomous deal sourcing system that:
1. **Finds** potential sellers via CoStar searches
2. **Reaches out** via email drip campaigns
3. **Qualifies** via calls and conversations
4. **Packages** deals for handoff to Brian or lee-1031-x

## Agent Workflows

### 1. Search â†’ Extraction Flow
```
Trigger: New search created (manual or from lee-1031-x)

1. SOURCING-AGENT
   Input: criteria_json
   Output: strategy_summary, payloads_json
   Update: search.status = 'extracting'

2. COSTAR-QUERY (per payload)
   Input: payload from payloads_json
   Output: properties, companies, contacts
   Update: search.total_* counts

3. Update: search.status = 'ready'
```

### 2. Campaign Creation Flow
```
Trigger: User clicks "Create Campaign" on search

1. Create campaign record (draft)
2. Generate enrollments for each property/contact
   - Check exclusions (DNC, bounce, already contacted THIS property)
   - Flag warnings (contacted about OTHER properties)
3. OUTREACH-COPY-GEN agent
   Input: search criteria, sample properties
   Output: 3 email templates (subject + body)
4. User reviews, adjusts settings, approves
5. Campaign status â†’ 'active'
```

### 3. Email Sending Flow
```
Trigger: Campaign active, within send window

1. DRIP-CAMPAIGN-EXEC
   - Find enrollments ready for next email
   - Check send window, rate limits
   - Send email via Outlook
   - Update enrollment (sent_at, current_step)
   - Random delay before next
```

### 4. Reply Processing Flow
```
Trigger: New email synced from Outlook

1. Match to enrollment (by thread/email)
2. RESPONSE-CLASSIFIER
   Input: email body, context
   Output: classification, confidence, reasoning
3. Update inbox_message with classification
4. Auto-actions based on classification:
   - BOUNCE â†’ add to exclusions
   - DNC â†’ add to exclusions, stop enrollment
   - WANTS_OFFER/INTERESTED â†’ create deal
   - WANTS_TO_BUY â†’ create pending search
   - SCHEDULE_CALL â†’ create call task
   - BROKER_REDIRECT â†’ flag, stop enrollment
5. Notify user (dashboard, email?)
```

### 5. Call Prep Flow
```
Trigger: Call scheduled

1. CALL-PREP-AGENT
   Input: contact, property, email thread, loan data, CoStar data
   Output: call_prep_md (markdown doc)
   - Background on seller/property
   - Key questions to ask
   - Qualification checklist
   - Talking points
```

### 6. Post-Call Processing Flow
```
Trigger: User pastes call notes

1. CALL-NOTES-AGENT
   Input: call notes, deal context
   Output:
   - Updated qualification fields
   - Action items
   - Next steps
   - If qualified â†’ prompt to package
   - If not interested â†’ close deal
   - If wants to buy â†’ create pending search
```

### 7. Deal Packaging Flow
```
Trigger: User clicks "Package Deal"

1. DEAL-PACKAGER
   Input: deal + all context (property, contact, calls, emails, docs)
   Output:
   - investment_summary (narrative)
   - investment_highlights (5 bullets)
   - Formatted for lee-1031-x schema
2. User reviews, approves
3. Export to lee-1031-x OR hand to Brian
```
